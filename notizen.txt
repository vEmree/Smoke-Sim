Struct Cell wird benutzt, damit man ein großes Gitter an Zellen haben kann, ohne dass man tausende lose variablen runfliegen hat

Wenn man eine normale Druckverteilung hätte, zb pressure.grid[5] = 10, sagen wir es gibt die hälfte seines drucks an seinen nachbarn grid.pressure[6] ab
grid.pressure[6] hätte also 5, grid.pressure[7] dann 2.5 usw, aber es würde sich instant von links nach rechts teleportieren weil es so schnell
durchgehen würde 

Lösung: Double Buffering
Man erstellt zwei Gitter (current_Grid; next_Grid)

current_Grid dient nur dazu um Werte des aktuellen Frames herauszulesen.
next_Grid werden neue berechnete Werte gegeben, die im nächsten Frame eingesetzt werden

damit man current_Grid und next_Grid effizient kopieren kann ohne unnötig speicher zu besetzen, musste ich struct Cell mit einem Pointer versehen, die 
auf die jeweiligen Grids Zeigt.
Und damit das funktioniert musste ich von der 2D Matrizen Art auf eine 1D Matritze übergehen, 
Nun muss ich, statt jede Zelle mit current_Grid[x][y] ansteuern zu können, jede zelle mit current_Grid[y * 100 + x] ansteuern
zb (x = 53 | y = 20) -> 20 * 100 + 53 -> 2053 = angesteuerter Index

dementsprechend sind dann die benachbarten Zellen findbar mit:
int current_cell = y * 100 + x;
int cell_up = (y-1) * 100 + x;
int cell_down = (y+1) * 100 + x;
int cell_left = y * 100 + (x-1);
int cell_right = y * 100 + (x+1);

struct Cell* temp = current_grid;
current_grid = next_grid;
next_grid = temp;

diese Schritte dienen dazu die current und next Grids zu tauschen

Beschleunigungsberechnung:
Damit man einen Vektor bekommt muss man sich die unglechheiten der nachbarn angucken, 
X Achse:
Positiver Vektor -> Bewegung nach rechts
Negitiver Vektor -> Bewegung nach links
new_vx = current_Grid[index_left].pressure - current_Grid[index_right].pressure

Y Achse:
Oben fängt 0 an
Positiver Vektor -> Bewegung nach unten
Negitiver Vektor -> Bewegung nach oben
new_vy = current_Grid[index_up].pressure - current_Grid[index_down].pressure
